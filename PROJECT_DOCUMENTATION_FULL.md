# Skillo Frontend - Полная документация проекта

Дата актуализации: 1 марта 2026

## 1. О чем проект

**Skillo** - это web-платформа для развития IT-специалистов по полному циклу:

- выбор направления и дорожной карты;
- определение текущего уровня;
- ежедневная практика и тесты;
- прохождение тем и интервью-вопросов;
- подтверждение навыков (онлайн/офлайн);
- работа с вакансиями и реальными задачами;
- социальный контур (друзья, челленджи, сообщество, лидерборд);
- профиль с прогрессом, резюме и сертификатом.

Проект построен как SPA на Vue 3 с модульной feature-архитектурой.

## 2. Технологии и инструменты

### 2.1 Основной стек

- `Vue 3` (Composition API)
- `Pinia` (глобальное состояние)
- `Vue Router` (маршрутизация)
- `Vite` (dev/build)
- `TypeScript` (в ключевых модулях и типах)
- `Axios` (подключен как зависимость)

### 2.2 UI/UX и инфраструктурные инструменты

- Единая тема (`light/dark`) через CSS custom properties и `data-theme`.
- Глобальная дизайн-система в `src/assets/main.css`.
- Анимации появления блоков через `IntersectionObserver` (`useRevealOnScroll`).
- Адаптивная боковая навигация и закрепление на desktop (`useResponsiveSideNav`).
- Моковый API-слой на базе `src/shared/api/services/*`.
- Персистентность пользовательских данных в `localStorage`.

## 3. Архитектура проекта

### 3.1 Структура

```text
src/
  app/
    layout/          # общий layout, navbar, side nav, footer, composables
    router/          # маршруты приложения
  features/          # бизнес-модули (views, stores, api)
  shared/
    api/             # mock api client + services + runtime/storage
    mocks/           # mock-данные
    types/           # контракты и общие типы
    ui/              # общие UI-компоненты (например, AmbientMotionLayer)
  assets/
    main.css         # глобальная тема и стили
```

### 3.2 Принципы

- Feature-based разделение по доменам.
- Ленивая загрузка страниц через `import()` в роутере.
- Разделение API-оберток по фичам (`features/*/api`) и shared-сервисов (`shared/api/services`).
- Вью-компоненты отвечают за orchestration UI, бизнес-данные хранятся в stores/API-слое.

## 4. Маршруты и функциональные разделы

| Маршрут | Раздел | Что умеет |
|---|---|---|
| `/` | О проекте | Длинная лендинг-страница с hero, секциями по всем разделам, визуальными блоками, hover/ambient эффектами и sparkle trail |
| `/login` | Вход | Авторизация, валидация полей, переход в onboarding или основной раздел |
| `/register` | Регистрация | Создание аккаунта, проверка паролей, переход в onboarding или основной раздел |
| `/onboarding` | Онбординг | Выбор интересов и первичная настройка профиля обучения |
| `/roadmaps` | Дорожные карты | Коллекция направлений, прогресс, удаление/добавление направлений, AI-конструктор пользовательских треков |
| `/roadmaps/:id/assessment` | Оценка roadmap | Быстрый тест для определения уровня и добавления направления в коллекцию |
| `/roadmaps/:id` | Детали roadmap | Два режима отображения: дерево навыков и классический список, прогресс по темам |
| `/topics/:id` | Тема | Теория, тест с таймером, подсчет результата, сохранение прогресса |
| `/topics/:id/interview` | Интервью | Список интервью-вопросов и ответов по теме |
| `/daily-tasks` | Ежедневные задания | Тест дня по каждому направлению + общий тест дня, фильтры, история, очки |
| `/skill-levels` | Определение уровня | Расширенная оценка (теория + письменные ответы), расчёт уровня от Junior до Senior |
| `/skill-verification` | Подтверждение навыков | Запись на online/offline слот, управление бронированиями, отметка прохождения, сертификат ID |
| `/vacancies` | Вакансии | Каталог вакансий, метаданные, мини-лидерборд по задачам, переход к подготовке |
| `/vacancies/:id/preparation` | Подготовка к вакансии | Обзор, Q&A, реальные задачи с отправкой решений, тест, локальный лидерборд |
| `/company` | Кабинет компании | CRUD вакансий, управление тестовыми задачами, кандидаты, отправка приглашений |
| `/friends` | Друзья и IT map | Добавление/удаление друзей, глобальная карта прогресса, челленджи и уведомления |
| `/community` | Сообщество | Публикации с модерацией, лайки, комментарии, лента и личная очередь модерации |
| `/leaders` | Лидеры | Рейтинг, фильтры по стране/городу/вузу, карточка пользователя |
| `/profile` | Профиль | Прогресс и активность, уровень по направлениям, резюме/сертификат, аналитика |

## 5. Что проект умеет сейчас (по факту реализации)

1. Работает как единая образовательная и карьерная экосистема с навигацией между всеми этапами.
2. Хранит прогресс пользователя локально и восстанавливает его между сессиями.
3. Поддерживает персонализацию через коллекцию roadmap, индивидуальные уровни, ежедневные задания, пользовательские AI-треки и социальные сценарии.
4. Поддерживает карьерные сценарии: подготовка к вакансии с задачами, лидерборды, подтверждение навыков с сертификатом, профиль и CV-сущности.
5. Предоставляет современный UI: фиксированный прозрачный navbar, responsive side nav, светлая и темная тема, анимации появления секций и интерактивные эффекты на странице проекта.

## 6. Состояние, данные и персистентность

### 6.1 Основные Pinia stores

- `auth` - токен, пользователь, first-login флаг.
- `roadmaps` - коллекция roadmap, уровни, прогресс.
- `topic-progress` - результаты тестов по темам.
- `daily-tasks` - задания дня, квизы, очки, напоминания.
- `skillLevels` - результаты определения уровня.
- `community` - посты, лайки, комментарии, модерация.

### 6.2 Ключевые localStorage-ключи

| Ключ | Назначение |
|---|---|
| `token` | auth token |
| `user_roadmap_ids` | выбранные дорожные карты пользователя |
| `user_roadmap_levels` | уровень пользователя по направлениям |
| `user_roadmap_progress` | прогресс по roadmap |
| `topic_test_results` | результаты тестов тем |
| `daily_tasks_db` | ежедневные задания и статусы |
| `daily_tasks_reminder_dismissed` | скрытие баннера daily reminder |
| `user_skill_levels_v1` | результаты раздела определения уровня |
| `skill_verification_bookings_v1` | записи на подтверждение навыков |
| `community_posts_v1` | посты сообщества и модерация |
| `custom_ai_tracks` | пользовательские AI-треки |
| `roadmap_view_mode` | режим отображения roadmap (tree/list) |
| `skilo-theme` | выбранная тема интерфейса |
| `skilo-desktop-nav-visible` | состояние боковой навигации на desktop |

Дополнительно mock-слой использует локальные ключи для хранилища пользователей, друзей, заявок, вакансий, сабмитов и кандидатов.

## 7. API и интеграционный слой

### 7.1 Текущий режим

- В проекте включен mock-runtime (`USE_MOCK = true` в `src/shared/api/shared/runtime.ts`).
- Данные имитируют серверные задержки (`delay(...)`), ответные DTO и контракт.

### 7.2 Контракт

- Единый контракт API описан в `src/shared/types/api-contract.ts`.
- Есть reference моделей моков: `src/shared/types/api-requests-reference.ts`.
- Shared API-композиция: `src/shared/api/client.ts` + сервисы в `src/shared/api/services/*`.

Покрытые домены API:

- auth
- roadmaps/progress/activity
- interview questions
- vacancies + task submissions + leaderboards
- social (friends, IT map, challenges)
- company cabinet (vacancy/task/candidate management)

- `^20.19.0` или `>=22.12.0`

## 10. Текущие ограничения

1. Backend сейчас имитируется mock-сервисами, нет реального API-подключения в production-режиме.
2. Нет выделенного e2e/unit тестового слоя в репозитории.
3. Часть бизнес-логики пока находится на клиенте и в localStorage.
4. Для production понадобится серверная авторизация и централизованное хранение прогресса.
